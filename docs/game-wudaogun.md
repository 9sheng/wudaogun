# 五阶精策棋 (5x5 Strategy Game) 详细设计文档 v2.0

## 1. 需求梳理与整理

### 1.1 基础设定
- **棋盘规模**：5x5 网格，棋子置于线的**交点**上（共 25 个落子位）。
- **棋子双方**：黑方、白方，交替行动。
- **成形规则（核心逻辑）**：
  所有斜线判定必须满足**两端点均在棋盘边缘**（行或列索引包含 0 或 4）：
    1. **3斜**：同色 3 子斜向连线，端点在边上。
    2. **4斜**：同色 4 子斜向连线，端点在边上。
    3. **通天 (5斜)**：同色 5 子斜向连线，端点在边上。
    4. **大棍**：同色 5 子横向或纵向连线（必触及两端边界）。
    5. **方**：同色 4 子组成 $2 \times 2$ 的小正方形。

### 1.2 游戏阶段
- **阶段 1 (下子阶段)**：双方交替落子直到满 25 处。成形时可标记对方一个棋子为“死亡”（掐子）。死子暂时不出场，且不参与后续成形判定。
- **阶段 2 (走子阶段)**：落子结束，移除所有标记为“死亡”的棋子。玩家轮流移动己方棋子至相邻空位。成形即立即提掉对方一子。
- **僵局处理**：若某方轮次中无路可走，需“自掐”一颗己方棋子（献祭移除），然后换手。

## 2. 交互与 UI 设计

### 2.1 5秒竞技窗口机制 (核心特色)
为了模拟真实对弈中的“眼力”竞争，设计如下逻辑：
- **计时启动**：玩家落子或走子动作完成后，立即启动 **5 秒倒计时**。
- **掐子触发**：玩家必须在倒计时结束前发现自己成形，并点击 **“我要掐子”** 按钮。
- **换手/错过判定**：
    - 若 5 秒内未点击按钮：系统自动换手，该轮次成形机会作废。
    - 若点击按钮：计时器停止。系统校验成形，若属实则进入掐子状态；若误判，则直接强制换手。

### 2.2 成形提醒辅助
- **设置开关 (Toggle)**：
    - **开启提示**：当检测到成形时，对应棋子会产生脉冲高亮或动画效果，引导玩家点击。用红色线段连接成型棋子。
    - **关闭提示（竞技模式）**：成形时无任何反馈，全凭玩家自主观察。
- **UI 组件**：
    - **倒计时进度条**：在状态栏下方展示红色递减进度条。
    - **状态反馈**：显示“窗口判定中...”或“请掐子”等明确指令。
