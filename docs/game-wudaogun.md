# 五导棍 (Wudaogun) 详细设计文档 v3.0

## 1. 基础设定

- **棋盘规模**：5×5 网格，棋子置于交点上（25 个落子位）。坐标 A–E（列）× 1–5（行）。
- **棋子双方**：黑方 (`B`) / 白方 (`W`)，交替行动。默认黑先，可切换白先。
- **移动规则**：仅上下左右正交移动一格（不含对角线）。

### 1.1 成形规则

所有斜线判定要求**两端点均在棋盘边缘**（行或列索引为 0 或 4）。

| 成形类型 | 代码标识 | 说明 |
|:---------|:---------|:-----|
| 三斜 | `diag3` | 同色 3 子斜向连线，两端触边 |
| 四斜 | `diag4` | 同色 4 子斜向连线，两端触边 |
| 通天 | `diag5` | 同色 5 子斜向连线，两端触边 |
| 大棍 | `line5` | 同色 5 子横向或纵向满行/满列 |
| 方   | `square` | 同色 4 子组成 2×2 正方形 |

斜线由 `Formation.DIAG_LINES` 预计算并去重。每次成形判定调用 `Formation.findAll()`，新成形通过 `Formation.findNew()` 与上一轮成形集合比对得出。

### 1.2 掐子目标选择

`Formation.pinchTargets()` 优先返回对方**不在任何成形中**的棋子；若对方所有棋子均在成形中，则全部可选。

## 2. 游戏阶段

### 2.1 下子阶段 (`PHASE_PLACE`)

- 双方交替落子至空位，直到 25 格填满（`placedCount >= 25`）。
- 成形时进入掐子流程：对方棋子被标记为"死亡"（`D` + 颜色，如 `DB`/`DW`），留在棋盘但不参与后续成形判定。
- 每个新成形授予 1 次掐子机会（`pinchesRemaining = newFormations.length`）。

### 2.2 阶段过渡

填满后自动进入走子阶段：
1. 显示 2 秒过渡动画（"⚔️ 走子阶段"）。
2. 调用 `Game.removeDeadPieces()` 移除所有死子。
3. 重新计算双方成形，检查是否有合法走步。

### 2.3 走子阶段 (`PHASE_MOVE`)

- 轮流移动己方棋子至相邻空位（正交方向）。
- 成形后立即提掉对方棋子（直接从棋盘移除，`board[r][c] = null`）。
- **僵局处理**：若当前方无合法走步，进入 `STATE_WAIT_SACRIFICE`，必须献祭己方一子。
- **胜负判定**：任一方棋子数归零即判负（`checkWin()`）。

### 2.4 游戏状态机

```
STATE_WAIT_ACTION     → 等待落子/走子
STATE_WAIT_PINCH_CLAIM → (保留，当前未使用)
STATE_WAIT_PINCH_SELECT → 选择掐子目标
STATE_WAIT_SACRIFICE   → 无路可走，选择献祭
```

## 3. 计时与掐子窗口

### 3.1 提示模式（`showHints = true`）

- 成形后自动进入 `STATE_WAIT_PINCH_SELECT`，启动 **10 秒**倒计时。
- 成形棋子以红色线段/矩形高亮，可掐目标以红色虚线圆圈标示。
- 玩家可点击掐子按钮暂停计时（`pauseTimer()`），然后选择目标。
- 超时则 `Game.expireClaim()` 放弃掐子机会并换手。

### 3.2 竞技模式（`showHints = false`）

- 无成形高亮，无掐子目标提示。
- 每次落子/走子后均启动 10 秒盲计时（`startBlindTimer()`），期间棋盘不可操作。
- 有成形时启动正常掐子计时，但不显示成形信息。
- 掐子按钮显示"⏳ 掐子结束，换手"。

### 3.3 多掐模式

`setMultiPinch(on)` 控制。关闭时每次成形最多掐 1 子（`pinchesRemaining` 强制为 1）。

## 4. AI 系统

### 4.1 引擎注册

`AI.get(level)` 返回对应引擎，统一接口：`choosePlace(g)`, `chooseMove(g)`, `choosePinch(g)`, `chooseSacrifice(g)`。

| 级别 | 引擎 | UI 标签 | 算法 |
|:-----|:-----|:--------|:-----|
| random | `AIRandom` | — | 随机选择 |
| greedy | `AIGreedy` | 🤖 初级 | 单步贪心 + 评估函数 |
| mcts | `AIMCTS` | 🤖 中级（默认） | 蒙特卡洛树搜索，2s 时间预算 |
| minimax | `AIMinimax` | 🤖 高级 | Alpha-Beta 剪枝 + 走法排序 |
| negamax | `AINegamax` | — | 迭代加深 Negamax，3s 时间预算 |

### 4.2 评估函数 (`AIEval`)

评估维度及权重：

- **棋子数差**：走子阶段 ×50，下子阶段 ×15；死子差 ×25
- **成形得分**：通天/大棍 80，四斜 50，三斜/方 30；对方成形 ×1.2 惩罚
- **威胁计数**：差 1 子成形的线路（`countThreats`），己方 ×2.5，对方 ×2.8
- **双重威胁**：≥2 个即时威胁时额外加分（己 ×20，对 ×22）
- **位置权重**：中心 5 分，次中心 4 分，角 3 分，边 2 分
- **连通性**：相邻同色棋子对数 ×2
- **机动性**（走子阶段）：合法走步数 ×4；对方 ≤1 步时 +30 陷阱奖励

### 4.3 AI 掐子动画

AI 掐子分两阶段：先显示 1 秒红色十字准星（`target`），再执行掐子并显示 1 秒移除动画（`removed`）。AI 掐子计时条为 2 秒。

## 5. 渲染与交互 (`Board`)

### 5.1 Canvas 棋盘

- 画布尺寸：`MARGIN(50) × 2 + CELL(90) × 4 = 460px`
- 棋子半径：`CELL × 0.38 ≈ 34px`
- 木纹渐变背景，径向渐变棋子，带阴影和高光

### 5.2 视觉反馈

| 元素 | 效果 |
|:-----|:-----|
| 最后落子 | 蓝色半透明光圈 |
| 选中棋子 | 绿色光环 + 可移动位置绿色圆点 |
| 成形高亮 | 红色线段连接（方为矩形） |
| 掐子目标 | 红色虚线圆圈 |
| 死子 | 灰色棋子 + 红色 X |
| 掐子动画 | 红色十字准星 → 红色扩散环 + X |
| 光标 | 动态生成当前方颜色的棋子光标 |

### 5.3 UI 控件

- **状态栏**：阶段标签 + 当前方 + 状态消息
- **计分板**：黑/白棋子数 + 落子进度（`placedCount/25`）
- **掐子按钮**：集成计时条，点击暂停计时或直接掐子
- **控制按钮**：悔棋（↩）、认输（🏳）、新局（🔄）
- **下拉选择**：AI 级别（双人/初级/中级/高级）、先手选择（黑先/白先）
- **开关**：成形提示、音效

### 5.4 悔棋

`undoMove()` 清除计时器，恢复上一步快照。对 AI 对局会连续撤销两步（AI + 玩家）。

---

## 6. 音效系统 (`Sound`)

基于 Web Audio API 合成，无外部文件。首次用户交互时 `unlock()` 解锁 AudioContext。

| 事件 | 函数 | 效果 |
|:-----|:-----|:-----|
| 落子/走子 | `place()` | 600Hz 正弦波 0.1s |
| 掐子 | `pinch()` | 300Hz→200Hz 方波 |
| 成形 | `formation()` | C5-E5-G5 三音上行 |
| 获胜 | `win()` | C5-E5-G5-C6 四音上行 |
| 献祭 | `sacrifice()` | 180Hz 锯齿波 0.3s |
| 计时 | `tick()` | 1000Hz 正弦波 0.03s |

## 7. 文件结构

```
index.html              # 入口页面，加载所有脚本
src/
  css/style.css         # 响应式 UI 样式（暗色主题，支持移动端）
  js/
    formation.js        # 成形检测（斜线预计算、方、大棍）
    game.js             # 核心状态机（落子、走子、掐子、献祭、胜负）
    board.js            # Canvas 渲染、事件处理、计时器、AI 调度
    sound.js            # Web Audio 音效合成
    ai/
      eval.js           # 共享评估函数
      random.js         # 随机 AI
      greedy.js         # 贪心 AI
      mcts.js           # 蒙特卡洛树搜索
      minimax.js        # Alpha-Beta 极小化极大
      negamax.js        # 迭代加深 Negamax
      ai.js             # AI 工厂（引擎注册与选择）
  test/test.js          # 单元测试
docs/
  game-wudaogun.md      # 本文档
```
